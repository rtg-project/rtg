The name "rtg" is derived from `relational to graph`, since its main focus is to provide a bridge between relational databases (SQL) and graph API (GraphQL).

Rtg acts translate GraphQL requests to SQL queries.

## Overview

```jsx
import { query } from '@rtg-project/client'

export async function getServerSideProps() {
  return {
    props: {
      users: await query(gql`{ users { id } }`)
    }
  }
}
```

## Features

With just one single line of code, you can simplify the logic of data fetching in your project, and also have all these amazing features out-of-the-box:

- **Fast**, **lightweight** and **reusable** queries
- TypeScript ready
- Rust ready
- React Native

And lot [more](/docs/getting-started).
